# Contribution Guidelines

Thank you for considering contributing to **LocalConnect**! We're excited to have you join our community. To maintain our code quality and facilitate a smooth collaboration, please follow these contribution guidelines:

## Code Standards
- Follow the [PEP 8](https://www.python.org/dev/peps/pep-0008/) style guide for Python code.
- Use meaningful variable and function names to improve code readability.
- Ensure code is well-documented with comments where necessary.

## Commit Messages
When making commits, please follow these guidelines:
- Use the present tense (e.g., "Add feature" instead of "Added feature").
- Be specific about the changes made. Include context when necessary.
- Use the format: `type: subject` (e.g., `fix: correct typo in README`).

## Testing Requirements
- Ensure that your code changes include appropriate unit tests and functional tests.
- Run all existing and new tests before making your pull request to ensure nothing is broken.
- Refer to our testing documentation for guidance on how to write and run tests.

## Pull Request Procedures
- Include a clear title and description for your pull request.
- Reference any related issues using `#IssueNumber` in the description.
- Make sure your pull request is targeting the correct branch.
- Be responsive to any feedback during the review process, and be prepared to make changes.

By following these guidelines, you help us to keep **LocalConnect** a healthy and enjoyable project to contribute to. Thank you!
